Design Recommendation: Approach A

Approach A best resolves the current ambiguity with minimal cognitive load. It keeps row-level context (variant name + value + allocation) tightly grouped, replaces edge-to-edge bars with constrained indicators that no longer read as loading states, and preserves fast scanability for both marketers and developers. Approach B is excellent for aggregate distribution but requires extra eye movement to map segments back to rows. Approach C is strongest for engineering density, but it feels heavier and less approachable in mixed product/marketing workflows.
